Operators

We spent a lot of last class discussing data types, but didn't really talk about what we can do with those data types. Lets talk about operators.
Some operators you will see often include:
  • Mathmatical
    • = (equals, assignment)
    • + (addition)
    • - (subtraction)
    • * (multiplication)
    • / (division)
    • % (modulus)
    • +=
    • -=
    • *=
    • /=
  • Comparison
    • == (loose comparison)
    • === (strict comparison)
    • != (loose not equal comparison)
    • !== (strict not equal comparison)
    • < (less than)
    • > (greater than)
    • <= (less than or equal to
    • >= (greater than or equal to
  • Logical
    • && (and)
    • || (or)

Control Flow

Control flow simply means controlling what code actually gets executed in your program.
The most basic form of control flow are if / else statements.
    
//simple if statement
if (true) {

    console.log("yup");

}

//if / else, if the first condition is true, run the first
//block, else run the second block
if (false) {

    alert("it's true");

} else {

    alert("it's false");

}

//if else if statments, the last else is our "fallback"
if (condition1) {


} else if (condition2) {


} else {


}

Operators & Conditions in Practice

Who's practiced more?

Jack has been writing javascript for 4 years. He spends 4 hours every day writing javascript. June has been writing javascript for 6 years, but only spends about 2.5 hours a day coding. Who has had more time at the keyboard? 
    
var daysInYear = 365;

var jackYears = 4;
var juneYears = 6;

var jackHoursPerDay = 4;
var juneHoursPerDay = 2.5;

var jackTotalHours = (jackHoursPerDay * daysInYear) * jackYears;
var juneTotalHours = (juneHoursPerDay * daysInYear) * juneYears;

if (jackTotalHours > juneTotalHours) {
    console.log("jack has more time at keyboard");
} else {
    console.log("june has more time at keyboard");
}
